## Decisoes-chave tomadas em 25/05 (timeline cronologica)

### 1. Tentativa de "bulletproof flow" (REVERTIDA)
Entre 22/05 e 25/05 manha — introduzi 4 commits tentando blindar o connect com:
- proxy setado ANTES do /create
- retry global + budget tracker
- pre-check de proxy manual no backend
- QR fallback

**Resultado:** quebrou tudo. `no_pairing_code_after_poll` em 100% dos pareamentos.

### 2. Revert para deploy AmjndGdRE de 8/5 (commit 63680cf)
Confirmado via `vercel promote` + git reset que essa era a unica versao funcional. Voltei o codigo inteiro pra esse estado em commit `45e9df2`.

### 3. Cleanup de conversa self pos-pareamento (c030b77)
Notificacao "device linked" do WhatsApp virava conversa pendurada na inbox. Adicionei endpoint que resolve essa conversa automaticamente apos setup.

### 4. Loop infinito de "Connection successfully established!"
Origem: contato system Chatwoot **id=1 "Atendimento" (+123456)** com avatar `evolution-api-favicon.png`. Evolution forwarda msgs de status pra ele em loop.

**Fix:** PATCH /contacts/1 { blocked: true } no Chatwoot. Resolve pra todos os chips de uma vez. Reversivel.

### 5. VIPB01 zombie + SSH
Instance VIPB01 entrou em estado zombie (`state: open` mas qualquer acao retornava "Connection Closed"). API rejeitou delete 5x.
- Acessei SSH via Hetzner Console + rescue mode + chroot pra resetar senha root
- Boot pra rescue (2x) acabou nao sendo necessario — o restart do servidor durante o processo de SSH limpou o estado zombie naturalmente
- Apos servidor voltar de rescue, instance estava em `close` e foi deletada normalmente pela API

### 6. Fix do timing do /create (ae939ba) - **CRITICO**
`/instance/create` da Evolution as vezes retorna sem `qrcode.pairingCode` (Baileys ainda inicializando). 2-3s depois o code esta disponivel em `/instance/connect/{name}`. Fix: fallback automatico no `/api/chips/connect`. Sem isso, connect falhava em chips novos com frequencia alta.

### 7. UI improvements (814da6f + 9c93c78)
- Chips ordenados Online -> Connecting -> Close, alfabetico dentro de cada grupo
- Campo proxy manual pre-preenchido com credencial marketbet
